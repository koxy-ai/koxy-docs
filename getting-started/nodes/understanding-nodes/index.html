Understanding nodes

What is a node​

A node is a single action that can be performed in your flow. Your flow is built up of different nodes, each with its own functionality and tasks.

Add a node​

Go to your cloudspace's flows, and open your flow's builder to start adding nodes to your flow. you can click on the plus icon in the top right corner to open a list of all available nodes to add from, once the nodes list is open you can choose any node you want to add, configure its options and click on Done.

The Koxy simulation system will always start running from the start1 node, so when you add your first node, so you need to notice few ideas:

  1. The start1 node will be always built-in with any flow you create.

  2. If you delete the start1 node the flow won't be readable by the Koxy simulation system.

  3. The start1 only has a name and a next property, to choose how your flow should start you would change the start1's next property to the trigger you want to start your flow from.

How nodes work​

Each node in a flow has a name, a type, an icon, a position, and a set of properties. The name of the node is used to identify it in the flow. The type of the node determines what the node does. The properties of the node control how the node behaves. All nodes share the following properties:

  • name: node names should be unique and can't be changed. This means that you cannot have two nodes with the same name in a flow. If you create a new node with the name of an existing one, it will replace the existing node.

  • type: defines the node's type (can't be changed).

  • next: this property defines the trigger that the node should trigger when it finishes running. It can be a node, a flow (syntax: koxy.flow.{FLOW_NAME}), or stop.

  • icon: the icon of the node (used by the Flows GUI Biulder).

  • x: the x position of the node (used by the Flows GUI Biulder).

  • y: the y position of the node (used by the Flows GUI Biulder).

Edit a node​

You will notice that every node has an options icon next to its name, if you click on it the configurations editor will open so you can edit your node and then click on Done to save your changes.

Nodes responses​

When a nodes finishes running, It will save its result as a response that can be accessed from other nodes with the following syntax koxy.res.{NODE_NAME}.

However, not all nodes support this feature like:

  • Boolean conditions.

  • Logger.

  • Delay.

Basic structure​

node_name: {
    type,
    icon,
    next,
    x,
    y,
    // other properties based on the node type
}

Delete a node​

You can delete a node from the navigator in the top-right corner (the edit icon).
