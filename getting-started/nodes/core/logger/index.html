Logger

The Logger node allows you to log and stream custom logs.

This node's behaviour changes based on the way that you're running your API.

How this node works​

  • Local runs: will just simply console.log the value giving to it.

  • Cloud runs: will stream live logs to the Koxy AI platform. so if you're running your API from the platform you will see the logs live, but when running your API from Koxy CLI using koxy cloudrun or using Koxy JS the logs will be returned inside the run's response.logs.

Add this node to your flow​

To add this node to your flow you should open the nodes list. check the flows builder documentation to learn more, and then You should fill the following properties:

  • name: node names should be unique and can't be changed.
  • log: the value you want to log.
  • Next: the trigger that the node should call after logging the value. click on the next input to get a list of available triggers.

Example​

In this example We will use the logger node to log the value of a specific parameter. sea Dynamic variables documentation for more information.

So first you need to give your node a name, in this example We will name it log1.

We will set the log input to this:

this is a parameter: koxy.get.{param1}

so if we send a request to the API with a parameters like this:

parameters: {
    param1: "parameter value"
}

our log1 node will log this:

this is a parameter: parameter value

learn more about parameters variables

Node structure​

The structure of the node we built before would be something like this:

log1: {
    type: "log",
    icon: "input",
    log: "this is a parameter koxy.get.{param1}",
    next: "stop",
    x: 0,
    y: 0
}
