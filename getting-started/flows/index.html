Flows

What is a flowâ€‹

A flow is a set of nodes or actions connected to each other is a specific order. You can think of flows as stages or functions in your API. actually your API is built up of flows.

Create a new flowâ€‹

To create a new flow, go to your cloudspace, click on the Flows icon, and then click on New flow. Give your flow a name and you've just created a new flow.

Flow names should be unique and cannot be changed. This means that you cannot have two flows with the same name. If you create a new flow with the name of an existing flow, it will replace the existing flow.

Your cloudspace can have an unlimited number of flows. We recommend giving your flows clear names to keep your API organized.

Connecting flowsâ€‹

You can connect flows together by setting a node's trigger to be another flow. This means that the Koxy simulation system will run the triggered flow next.

You can set the trigger to be a flow manually by setting the Next property like this: koxy.flow.{FLOW_NAME}. Replace FLOW_NAME with the name of the flow you want to trigger.

Every flow comes with a start1 node that should not be deleted.

How to change your default flowâ€‹

When you run your API It will start running from the main flow by default but you can configure the entry option to choose the flow that your API should start from.

Koxy CLIâ€‹

you can edit the koxy.config.json file in your cloudspace directory and change the entry option to any flow you want your API to start from. here is an example of how your koxy.config.json should look like this:

{
    "cloudspace": CLOUDSPACE_TOKEN,
    "local": {
        "entry": "main",
        "logs": true,
        "responseType": "json"
    },
    "cloud": {

        "source": "flows"
    }
}

Koxy JS libraryâ€‹

you can set the entry option in your configurations. here is an example:

import { koxyRun } from 'koxy-js';

const configs = {
    cloudspace: CLOUDSPACE_TOKEN
    entry: "my_flow"
};

const run = await koxyRun(configs);

Check the Koxy JS library to learn more.

This feature adds a lot of flixibility to your API and for developers who want to build versatile and advanced APIs that can be used for a lot of use cases.

Can I build my API in one flow ?â€‹

It's possible for you to build your API in one flow, but however It's not receommended.

If You want to build so many APIs and you don't want to make multiple cloudspaces, You can build multiple APIs in one cloudspace by acting with every flow as a dedicated API.

This can be useful for developers with low budget or who don't want to buy a Pro plan for every API they make.
